Comment les utilisateurs créent un nouveau compte

Après qu'un administrateur a ajouté un utilisateur à un cas : * si l'utilisateur est nouveau sur caseactive, l'utilisateur doit terminer un workflow pour créer un compte et accepter l'invitation à accéder au cas. * Si l'utilisateur a un compte caseactive existant, l'utilisateur doit accepter l'invitation pour accéder au dossier.

Cet article décrit les étapes qu'un utilisateur doit suivre pour créer son compte.

Commencer

Un utilisateur peut être ajouté à une application caseactive de deux manières

  • Par l'administrateur : si l'administrateur crée un utilisateur via l' interface utilisateur de la plateforme ou l' outil d'importation , l'administrateur doit envoyer à l'utilisateur un lien vers account.caseactive.com pour procéder à la connexion.
  • Par utilisateur : Si l'utilisateur a créé un compte à l'aide d'un lien d'inscription , l'utilisateur sera automatiquement redirigé vers account.caseactive.com pour procéder à la connexion.

Se connecter

Lorsque l'utilisateur se connecte à son compte CaseActive pour la première fois, la plateforme invite l'utilisateur à :

  • ouvrez leur compte de messagerie, puis copiez/collez le code de sécurité à six chiffres
  • remplir un profil de compte

image

Une fois que l'utilisateur a cliqué sur METTRE À JOUR, la page Applications s'affiche. La page affiche le nom de l'application (nom de l'entreprise), le type d'utilisateur (rôle), le statut et un bouton INVITATION.

image

Pour accéder à l'application, l'utilisateur doit accepter l'invitation. L'utilisateur doit :

  1. Cliquez sur le bouton INVITATION.
  2. Vérifiez les autorisations.
  3. Cliquez sur l'onglet Notifications pour revoir et modifier (facultatif) les termes.
  4. Cliquez sur APPROUVER L'ACCÈS.

image

Lorsque l'utilisateur approuve, la plateforme affiche un bouton APP sur la page utilisateur. Ce bouton permet à l'utilisateur d'accéder aux cas attribués.

Pour modifier les paramètres utilisateur, l'utilisateur clique sur les points de suspension.

image

L'utilisateur peut : * supprimer l'accès à l'application * modifier les paramètres de notification

image

Une fois que l'utilisateur a terminé le workflow, le nom d'utilisateur apparaît dans la liste des utilisateurs de l'administrateur. La liste affiche tous les utilisateurs, connectés ou non.
