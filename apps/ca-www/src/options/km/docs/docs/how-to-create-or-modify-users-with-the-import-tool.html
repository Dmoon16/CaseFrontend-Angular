របៀបបង្កើត ឬកែប្រែអ្នកប្រើប្រាស់ដោយប្រើឧបករណ៍នាំចូល

អត្ថបទនេះបង្ហាញអ្នកគ្រប់គ្រងពីរបៀបបង្កើតអ្នកប្រើប្រាស់ដោយប្រើឧបករណ៍នាំចូល។

ការបើកឧបករណ៍នាំចូល

អ្នក​គ្រប់គ្រង​អាច​ប្រើ​ឧបករណ៍​នាំចូល​ដើម្បី​ផ្ទុក​អ្នក​ប្រើ​ថ្មី​ជា​ច្រើន និង​កែប្រែ​កំណត់ត្រា​អ្នក​ប្រើ​ដែល​មាន​ស្រាប់។

  1. ចូលទៅគណនីអ្នកគ្រប់គ្រង។
  2. ស្វែងរកទីតាំងកម្មវិធីនៅលើទំព័រកម្មវិធី ហើយចុចប៊ូតុង ADMIN ។
  3. នៅលើផ្ទាំងគ្រប់គ្រង ចុចលើផ្ទាំង ការរួមបញ្ចូល ហើយបន្ទាប់មក IMPORTS នៅលើម៉ឺនុយរង។
  4. ចុចប៊ូតុង NEW ។ ប្រអប់បន្ថែមការនាំចូលបង្ហាញនៅក្នុងផ្ទាំងកម្មវិធីរុករកថ្មី។

ការបង្កើតកំណត់ត្រា

បន្ទាប់ពីប្រអប់បន្ថែមការនាំចូលបើក៖

  1. ចុចលើវាល ប្រភេទ ហើយជ្រើសរើសជម្រើសមួយ (សូមមើលខាងក្រោម)។ តំណបង្ហាញ។
  2. ចុចលើតំណ ទាញយកគំរូ ។ ឯកសារ CSV ទាញយកទៅឧបករណ៍របស់អ្នក។

បន្ទាប់បើកឯកសារហើយបន្ថែមកំណត់ត្រា។ ចុច ទីនេះ ដើម្បីស្វែងយល់ពីរបៀបប្រើទិន្នន័យទម្រង់ក្នុងឯកសារ CSV ។

ប្រភេទ

នៅពេលអ្នកចុចលើវាល ប្រភេទ ម៉ឺនុយទម្លាក់ចុះនឹងបង្ហាញ។ ជ្រើសរើសជម្រើសមួយក្នុងចំណោមជម្រើសពីរសម្រាប់ការនាំចូលភាគច្រើន។

:exclamation: ចំណាំ ម៉ឺនុយទម្លាក់ចុះមានជម្រើសជាច្រើន ប៉ុន្តែមានតែពីរប៉ុណ្ណោះដែលទាក់ទងនឹងឧបករណ៍នាំចូល។ ជម្រើសនីមួយៗមានឯកសារ CSV គំរូផ្សេងគ្នា។

បន្ថែមអ្នកប្រើប្រាស់

  • ជ្រើសរើសជម្រើសនេះសម្រាប់ការផ្ទុកឡើងភាគច្រើនសាមញ្ញនៃកំណត់ត្រាអ្នកប្រើប្រាស់។
  • ចុចតំណទាញយកគំរូដើម្បីបើកឯកសារ CSV ដែលមានចំណងជើងជួរឈរសម្រាប់ប្រភេទបន្ថែមអ្នកប្រើប្រាស់នាំចូល។

កែប្រែអ្នកប្រើប្រាស់

  • ជ្រើសរើសជម្រើសនេះដើម្បីធ្វើបច្ចុប្បន្នភាពទិន្នន័យសម្រាប់អ្នកប្រើប្រាស់បច្ចុប្បន្ន។
  • ចុចតំណទាញយកគំរូដើម្បីបើកឯកសារ CSV ដែលមានចំណងជើងជួរឈរសម្រាប់ កែប្រែអ្នកប្រើប្រាស់ ប្រភេទនាំចូល។

កំពុងបង្ហោះឯកសារ

អ្នកអាចផ្ទុកឡើងឯកសារភ្លាមៗ ឬត្រឡប់ទៅប្រអប់បន្ថែមការនាំចូលនៅពេលក្រោយ ហើយបំពេញជំហានទាំងនេះ។

  1. បើកប្រអប់ បន្ថែមការនាំចូល។
  2. បញ្ចូលឈ្មោះតែមួយគត់សម្រាប់ឯកសារ CSV ។
  3. ចុចរូបតំណាងបូក (+) ដើម្បីបង្ហោះឯកសារពីឧបករណ៍របស់អ្នក។
  4. ចុច​លើ​ប្រអប់ Type ហើយ​ជ្រើសរើស​ជម្រើស​មួយ។ (មើល​ខាងលើ)
  5. ចុច រក្សាទុក។

ដំណើរការ

អាស្រ័យលើចំនួនកំណត់ត្រានៅក្នុងឯកសារ និងការផ្ទុកប្រព័ន្ធ វេទិកាអាចចំណាយពេលច្រើននាទីដើម្បីដំណើរការការនាំចូល។

បរាជ័យ

ប្រសិនបើប្រតិបត្តិការនាំចូលបរាជ័យ វេទិកានឹងត្រឡប់សារដែលបរាជ័យ។ ក្នុងករណីភាគច្រើន មូលហេតុនៃការបរាជ័យគឺការធ្វើទ្រង់ទ្រាយទិន្នន័យ។

ដើម្បីមើលមូលហេតុនៃការបរាជ័យ៖
