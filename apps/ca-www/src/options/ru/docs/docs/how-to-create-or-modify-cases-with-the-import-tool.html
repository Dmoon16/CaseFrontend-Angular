Как создавать или изменять обращения с помощью инструмента импорта

В этой статье администраторам показано, как создавать или изменять существующие обращения с помощью инструмента импорта.

Открытие инструмента импорта

  1. Войдите в учетную запись администратора.
  2. Найдите приложение на странице приложений и нажмите кнопку ADMIN.
  3. На панели администратора щелкните вкладку «Интеграция», а затем «ИМПОРТ» в подменю.
  4. Нажмите кнопку НОВЫЙ. Диалоговое окно «Добавить импорт» отображается на новой вкладке браузера.

изображение

Создание кейсов

После открытия диалогового окна «Добавить импорт»:

  1. Щелкните поле Тип и выберите параметр (см. ниже). Отображается ссылка.
  2. Щелкните ссылку Загрузить образец. CSV-файл загружается на ваше устройство.

Затем откройте файл и добавьте дела.

Тип

Если щелкнуть поле Тип , появится раскрывающееся меню. Выберите один из двух вариантов импорта объемных ящиков.

  • Примечание . В раскрывающемся меню есть много параметров, но только два из них связаны с управлением регистром. Каждый вариант имеет отдельный образец CSV-файла.

Добавить дела

  • Выберите этот вариант для простой массовой загрузки обращений.
  • Щелкните ссылку «Загрузить образец», чтобы открыть файл CSV с заголовками столбцов для типа импорта «Создать обращения».

Изменить дела

  • Выберите этот параметр, чтобы обновить текущие дела.
  • Щелкните ссылку «Загрузить образец», чтобы открыть CSV-файл с заголовками столбцов для типа импорта «Изменить дела».

Загрузка файла

Вы можете загрузить файл сразу или вернуться к диалоговому окну «Добавить импорт» позже и выполнить эти шаги.

  1. Откройте диалоговое окно «Добавить импорт».
  2. Введите уникальное имя для CSV-файла.
  3. Нажмите значок плюса (+), чтобы загрузить файл с вашего устройства.
  4. Щелкните поле Тип и выберите параметр. (см. выше)
  5. Нажмите СОХРАНИТЬ.

Форматирование

Это обязательные поля и первые требования. Нажмите здесь, чтобы просмотреть детали форматирования.

tag_id (или case_ID) уникальный идентификатор (используйте буквенно-цифровые символы, допускаются дефисы)

case_status 0 = закрыто 1 = активно

date_opened месяц/день/год мм/дд/гггг

об уникальном описании дела (используйте буквенно-цифровые символы, допускаются дефисы)

Обработка

В зависимости от количества записей в файле и загрузки системы платформе может потребоваться несколько минут для обработки импорта.

изображение

По завершении импорта платформа корректирует записи обращений (т. е. добавляет или изменяет) на панели управления обращениями администратора.

Отказ

Если операция импорта завершается неудачно, платформа возвращает сообщение Failed. В большинстве случаев причиной сбоя является форматирование данных.

изображение

Чтобы увидеть причину сбоя:

  1. Щелкните Просмотр под заголовком ДЕЙСТВИЕ.
  2. На следующей странице наведите указатель мыши на значок «Не удалось». Во всплывающей подсказке отображается ошибка.
  3. Перейдите к файлу CVS, настройте и исправьте ошибку.
  4. Загрузите файл еще раз.
