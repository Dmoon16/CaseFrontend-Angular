Как назначать, изменять или удалять назначения пользователей с помощью инструмента импорта

В этой статье администраторам показано, как управлять пользователями дел с помощью инструмента импорта.

Администраторы могут:

  • назначать пользователей к делу
  • изменять пользователей, уже назначенных делу
  • удалить пользователей, назначенных для дела

Открытие инструмента импорта

  1. Войдите в учетную запись администратора.
  2. Найдите приложение на странице приложений и нажмите кнопку ADMIN.
  3. Перейдите на вкладку «Интеграция», а затем в подменю «ИМПОРТ».
  4. Нажмите кнопку НОВЫЙ. Откроется диалоговое окно «Добавить импорт».

Создание записей

Загрузите один или несколько файлов CSV. Используйте эти файлы для добавления, изменения или удаления пользователей обращения. После открытия диалогового окна «Добавить импорт»:

  1. Щелкните поле Тип.
  2. Прокрутите раскрывающееся меню и выберите вариант (см. ниже). Отображается ссылка.
  3. Щелкните ссылку Загрузить образец. CSV-файл загружается на ваше устройство.

Откройте файл и добавьте записи. Щелкните здесь, чтобы узнать, как использовать данные формата в файле CSV.

Параметры файла

Назначение пользователей к обращениям * Выберите этот параметр, чтобы добавить пользователей к обращению. * Оставьте ячейку case_role_order пустой. * Перейдите на страницу «Пользователи» и скопируйте идентификатор тега для этого человека. Если поле «Идентификатор тега» пусто, введите уникальный идентификатор, нажмите «Сохранить», а затем скопируйте идентификатор в файл CSV.

Загрузка файла

Вы можете загрузить файл сразу или вернуться к диалоговому окну «Добавить импорт» позже и выполнить эти шаги.

  1. Откройте диалоговое окно «Добавить импорт».
  2. Введите уникальное имя для CSV-файла.
  3. Нажмите значок плюса (+), чтобы загрузить файл с вашего устройства.
  4. Щелкните поле Тип и выберите параметр. (см. выше)
  5. Нажмите СОХРАНИТЬ.

Обработка

В зависимости от количества записей в файле и загрузки системы платформе может потребоваться несколько минут для обработки импорта.

изображение

По завершении импорта платформа корректирует пользовательские записи (т. е. добавляет, изменяет или удаляет) на странице «Обращения».

Отказ

Если операция импорта завершается неудачно, платформа возвращает сообщение Failed. В большинстве случаев причиной сбоя является форматирование данных.

изображение

Чтобы увидеть причину сбоя:

  1. Нажмите «Просмотр» под заголовком «Действие».
  2. На следующей странице наведите указатель мыши на значок «Не удалось». Во всплывающей подсказке отображается ошибка.
  3. Перейдите к файлу CVS, настройте и исправьте ошибку.
  4. Загрузите файл еще раз.
