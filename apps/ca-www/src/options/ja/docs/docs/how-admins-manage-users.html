管理者がユーザーを管理する方法

管理者は、アプリやケースにアクセスできるユーザーを作成、変更、または無効にします。ユーザーがプラットフォームに初めてログインするときは、プロファイル ワークフローを完了する必要があります。

  • 新しいユーザー プロファイル ワークフローの詳細については、ここをクリックしてください。

ユーザーの管理

次の表は、ユーザーを管理する 5 つの方法を示しています。方法については、リンクをクリックしてください。

方法 タイプ 行動 詳細
UI の使用 UI 追加＆修正 管理者ボタン > ユーザー
ケースの作成中およびユーザーをケースに割り当てる UI 追加 [管理] ボタン > [ケース] > [新規] ボタン > [ケースの作成] ページ
公開サインアップ リンクを使用してユーザーを自動作成 ユーザーアクション 追加 管理者ボタン > 設定 > システム設定ページ
インポートツール UI 追加＆修正 管理ボタン > 統合 > インポート
API の使用 (近日公開予定) API 追加＆修正 なし

通知

ユーザーがアプリに追加されても、プラットフォームはユーザーに通知を送信しません。ただし、管理者はケース割り当ての通知を有効にすることができます。
