管理者がユーザーを作成する方法

管理者は、アプリやケースにアクセスできるユーザーを作成、変更、または無効にします。ユーザーがプラットフォームに初めてログインするときは、プロファイル ワークフローを完了する必要があります。

  • 新しいユーザー プロファイル ワークフローの詳細については、ここをクリックしてください。

ユーザーの作成

次の表に、ユーザーを作成する 5 つの方法を示します。方法については、リンクをクリックしてください。

| |メソッド |タイプ |アクション |詳細 |新規ユーザーは通知を受け取りますか? | |
|-------------------------|---------------|------- ------|--------------------|---------------------- ------------------------|

UI の使用| UI |追加 & 変更 |管理ボタン > ユーザー |いいえ |

ケースの作成中にユーザーをケースに割り当てる| UI |追加 | [管理] ボタン > [ケース] > [新規] ボタン > [ケースの作成] ページ |はい、有効な場合 |

公開サインアップ リンクを使用してユーザーを自動作成|ユーザー アクション |追加 |管理者ボタン > 設定 > システム設定ページ |はい、ケースの自動作成が有効になっている場合 |

インポート ツール| UI |追加 & 変更 |管理ボタン > 統合 > インポート |いいえ |

API の使用 (近日公開予定) | API |追加 & 変更 |な |な |

通知

ユーザーがアプリに追加されても、プラットフォームはユーザーに通知を送信しません。ただし、管理者はケース割り当ての通知を有効にすることができます。
