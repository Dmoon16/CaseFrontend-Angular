Πώς οι διαχειριστές δημιουργούν χρήστες

Οι διαχειριστές δημιουργούν, τροποποιούν ή απενεργοποιούν τα άτομα που έχουν πρόσβαση σε εφαρμογές και υποθέσεις. Όταν ένας χρήστης συνδέεται στην πλατφόρμα για πρώτη φορά, πρέπει να ολοκληρώσει μια ροή εργασίας προφίλ.

  • Κάντε κλικ εδώ για να μάθετε περισσότερα σχετικά με τη ροή εργασίας του νέου προφίλ χρήστη.

Δημιουργία χρήστη

Ο παρακάτω πίνακας δείχνει πέντε τρόπους δημιουργίας χρήστη. Κάντε κλικ σε έναν σύνδεσμο για να μάθετε πώς.

| Μέθοδος | Τύπος | Δράσεις | Λεπτομέρειες | Ο νέος χρήστης λαμβάνει ειδοποίηση; |
|------------------------|---------------|------- ------|-------------------------------------------- ------------------------|

Χρήση διεπαφής χρήστη | UI | προσθήκη & τροποποίηση | Κουμπί διαχειριστή > Χρήστες | Όχι |

Κατά τη δημιουργία μιας υπόθεσης και εκχώρηση χρήστη σε περίπτωση | UI | προσθήκη | Κουμπί διαχειριστή > Υποθέσεις > Κουμπί Νέο > Δημιουργία σελίδας υπόθεσης | Ναι, εάν είναι ενεργοποιημένο |

Αυτόματη δημιουργία χρήστη χρησιμοποιώντας δημόσιο σύνδεσμο εγγραφής | Ενέργεια χρήστη | προσθήκη | Κουμπί διαχειριστή > Ρυθμίσεις > Σελίδα ρύθμισης συστήματος | Ναι, εάν είναι ενεργοποιημένη η αυτόματη δημιουργία θήκης |

Εργαλείο εισαγωγής | UI | προσθήκη & τροποποίηση | Κουμπί διαχειριστή > Ενσωματώσεις > Εισαγωγές | Όχι |

Χρήση API (προσεχώς) | API | προσθήκη & τροποποίηση | na | na |

Ειδοποιήσεις

Η πλατφόρμα δεν στέλνει στους χρήστες ειδοποίηση όταν προστίθενται σε μια εφαρμογή. Ωστόσο, οι διαχειριστές μπορούν να ενεργοποιήσουν τις ειδοποιήσεις για αναθέσεις υποθέσεων.
