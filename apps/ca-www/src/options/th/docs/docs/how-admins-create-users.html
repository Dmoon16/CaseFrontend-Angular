ผู้ดูแลระบบสร้างผู้ใช้อย่างไร

ผู้ดูแลระบบสร้าง แก้ไข หรือปิดใช้งานผู้ที่สามารถเข้าถึงแอปและกรณีต่างๆ เมื่อผู้ใช้เข้าสู่ระบบแพลตฟอร์มเป็นครั้งแรก พวกเขาจำเป็นต้องดำเนินการขั้นตอนการทำงานของโปรไฟล์ให้เสร็จสิ้น

  • คลิก ที่นี่ เพื่อเรียนรู้เพิ่มเติมเกี่ยวกับเวิร์กโฟลว์โปรไฟล์ผู้ใช้ใหม่

การสร้างผู้ใช้

ตารางด้านล่างแสดงห้าวิธีในการสร้างผู้ใช้ คลิกลิงก์เพื่อเรียนรู้วิธีการ

| วิธีการ | พิมพ์ | การกระทำ | รายละเอียด | ผู้ใช้ใหม่ได้รับการแจ้งเตือน? |
|-------------------------|------------------------|------- ------|----------------------|---------------------- ------------------------|

การใช้ UI | UI | เพิ่มและแก้ไข | ปุ่มผู้ดูแลระบบ > ผู้ใช้ | ไม่ |

ขณะสร้างเคสและกำหนดผู้ใช้ให้กับเคส | UI | เพิ่ม | ปุ่มผู้ดูแลระบบ > กรณีและปัญหา > ปุ่มใหม่ > สร้างหน้ากรณีและปัญหา | ใช่ ถ้าเปิดใช้งาน |

สร้างผู้ใช้อัตโนมัติโดยใช้ลิงก์ลงทะเบียนสาธารณะ | การกระทำของผู้ใช้ | เพิ่ม | ปุ่มผู้ดูแลระบบ > การตั้งค่า > หน้าการตั้งค่าระบบ | ใช่ หากเปิดใช้งานกรณีสร้างอัตโนมัติ |

เครื่องมือนำเข้า | UI | เพิ่มและแก้ไข | ปุ่มผู้ดูแลระบบ > การผสานรวม > นำเข้า | ไม่ |

การใช้ API (เร็วๆ นี้) | API | เพิ่มและแก้ไข | นา | นา |

การแจ้งเตือน

แพลตฟอร์มนี้จะไม่ส่งการแจ้งเตือนให้ผู้ใช้เมื่อมีการเพิ่มผู้ใช้ลงในแอป อย่างไรก็ตาม ผู้ดูแลระบบสามารถเปิดใช้งานการแจ้งเตือนสำหรับการกำหนดกรณีและปัญหาได้
