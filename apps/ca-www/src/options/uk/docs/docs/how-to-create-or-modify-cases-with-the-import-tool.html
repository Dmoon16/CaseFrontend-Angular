Як створити або змінити випадки за допомогою інструмента імпорту

У цій статті адміністраторам показано, як створювати або змінювати наявні випадки за допомогою інструмента «Імпорт».

Відкриття інструмента імпорту

  1. Увійдіть в обліковий запис адміністратора.
  2. Знайдіть програму на сторінці програм і натисніть кнопку АДМІНІСТРАТОР.
  3. На панелі адміністратора натисніть вкладку Інтеграція, а потім у підменю виберіть ІМПОРТ.
  4. Натисніть кнопку НОВИЙ. У новій вкладці веб-переглядача з’явиться діалогове вікно «Додати імпорт».

Створення кейсів

Після відкриття діалогового вікна «Додати імпорт»:

  1. Клацніть поле Тип і виберіть параметр (див. нижче). Відобразиться посилання.
  2. Натисніть посилання Завантажити зразок. Файл CSV завантажується на ваш пристрій.

Далі відкрийте файл і додайте справи.

Тип

Коли ви клацаєте поле « Тип », з’являється спадне меню. Виберіть один із двох варіантів масового імпорту випадків.

  • Примітка . У розкривному меню є багато параметрів, але лише два пов’язані з керуванням справою. Кожен варіант має окремий зразок файлу CSV.

Додайте справи

  • Виберіть цей варіант для простого масового завантаження випадків.
  • Натисніть посилання «Завантажити зразок», щоб відкрити файл CSV із заголовками стовпців для типу імпорту «Створити справи».

Змінити випадки

  • Виберіть цей параметр, щоб оновити поточні справи.
  • Натисніть посилання «Завантажити зразок», щоб відкрити файл CSV із заголовками стовпців для типу імпорту «Змінити випадки».

Завантаження файлу

Ви можете завантажити файл одразу або повернутися до діалогового вікна «Додати імпорт» пізніше та виконати ці дії.

  1. Відкрийте діалогове вікно «Додати імпорт».
  2. Введіть унікальну назву для файлу CSV.
  3. Натисніть значок плюса (+), щоб завантажити файл зі свого пристрою.
  4. Клацніть поле Тип і виберіть потрібний параметр. (Дивись вище)
  5. Натисніть ЗБЕРЕГТИ.

Форматування

Це обов’язкові поля та перші вимоги. Натисніть тут , щоб переглянути деталі форматування.

tag_id (або case_ID) унікальний ідентифікатор (використовуйте буквено-цифрові символи, допускаються дефіси)

case_status 0 = закрито 1 = активний

date_opened місяць/день/рік мм/дд/рррр

про унікальний опис справи (використовуйте буквено-цифрові символи, допускаються дефіси)

Обробка

Залежно від кількості записів у файлі та завантаження системи платформі може знадобитися кілька хвилин, щоб обробити імпорт.

малюнок

Після завершення імпорту платформа коригує записи випадків (тобто додає або змінює) на інформаційній панелі справ адміністратора.

провал

Якщо операція імпорту не вдається, платформа повертає повідомлення Не вдалося. У більшості випадків причиною збою є форматування даних.

малюнок

Щоб побачити причину збою:

  1. Натисніть Переглянути під заголовком ДІЯ.
  2. На наступній сторінці наведіть курсор миші на піктограму «Не вдалося». Підказка відображає помилку.
  3. Перейдіть до файлу CVS і виправте помилку.
  4. Завантажте файл ще раз.
