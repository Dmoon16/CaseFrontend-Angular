Cómo los administradores crean usuarios

Los administradores crean, modifican o deshabilitan quién puede acceder a aplicaciones y casos. Cuando un usuario inicia sesión en la plataforma por primera vez, debe completar un flujo de trabajo de perfil.

  • Haga clic aquí para obtener más información sobre el flujo de trabajo del nuevo perfil de usuario.

Creando un usuario

La siguiente tabla muestra cinco formas de crear un usuario. Haga clic en un enlace para saber cómo.

| Método | Tipo | Acciones | Detalles | ¿El nuevo usuario recibe una notificación? |
|-------------------------|---------------|------- ------|--------------------|------------------------------------- ------------------------|

Uso de la interfaz de usuario | interfaz de usuario | agregar y modificar | Botón de administración > Usuarios | No |

Al crear un caso y asignar un usuario al caso | interfaz de usuario | añadir | Botón Admin > Casos > botón Nuevo > página Crear caso | Sí, si está habilitado |

Creación automática de usuarios usando un enlace de registro público | Acción del usuario | añadir | Botón Admin > Configuración > página Configuración del sistema | Sí, si la creación automática de casos está habilitada |

Herramienta de importación | interfaz de usuario | agregar y modificar | Botón de administración > Integraciones > Importaciones | No |

Uso de API (próximamente) | API | agregar y modificar | na | na |

Notificaciones

La plataforma no envía una notificación a los usuarios cuando se agregan a una aplicación. Sin embargo, los administradores pueden habilitar las notificaciones para las asignaciones de casos.
