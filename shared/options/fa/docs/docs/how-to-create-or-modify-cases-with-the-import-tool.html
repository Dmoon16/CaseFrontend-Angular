نحوه ایجاد یا اصلاح موارد با ابزار Import

این مقاله به مدیران نحوه ایجاد یا اصلاح موارد موجود با ابزار Import را نشان می‌دهد.

باز کردن ابزار Import

  1. وارد حساب کاربری مدیریت شوید.
  2. یک برنامه را در صفحه برنامه ها پیدا کنید و روی دکمه ADMIN کلیک کنید.
  3. در پنل مدیریت، روی تب Integration و سپس IMPORTS در منوی فرعی کلیک کنید.
  4. روی دکمه NEW کلیک کنید. کادر محاوره‌ای افزودن واردات در یک برگه مرورگر جدید نمایش داده می‌شود.

img

ایجاد موارد

پس از باز شدن کادر محاوره ای Add Imports:

  1. روی فیلد Type کلیک کنید و یک گزینه را انتخاب کنید (به زیر مراجعه کنید). یک لینک نمایش داده می شود.
  2. روی لینک دانلود نمونه کلیک کنید. یک فایل CSV در دستگاه شما دانلود می شود.

بعد، فایل را باز کنید و موارد را اضافه کنید.

تایپ کنید

وقتی روی فیلد Type کلیک می کنید، یک منوی کشویی نمایش داده می شود. یکی از دو گزینه را برای واردات انبوه کیس انتخاب کنید.

  • توجه داشته باشید منوی کشویی گزینه های زیادی دارد، اما تنها دو مورد مربوط به مدیریت پرونده هستند. هر گزینه یک نمونه فایل CSV متفاوت دارد.

موارد را اضافه کنید

  • این گزینه را برای آپلود انبوه ساده موارد انتخاب کنید.
  • برای باز کردن یک فایل CSV با عناوین ستون‌ها برای نوع وارد کردن موارد، روی پیوند دانلود نمونه کلیک کنید.

موارد را اصلاح کنید

  • این گزینه را برای به روز رسانی موارد فعلی انتخاب کنید.
  • برای باز کردن یک فایل CSV با عناوین ستون برای نوع وارد کردن موارد Modify روی پیوند دانلود نمونه کلیک کنید.

در حال آپلود فایل

می‌توانید فوراً یک فایل را آپلود کنید یا بعداً به کادر محاوره‌ای افزودن واردات بازگردید و این مراحل را کامل کنید.

  1. کادر محاوره ای Add Imports را باز کنید.
  2. یک نام منحصر به فرد برای فایل CSV وارد کنید.
  3. برای آپلود فایل از دستگاه خود، روی نماد مثبت (+) کلیک کنید.
  4. روی فیلد Type کلیک کنید و یک گزینه را انتخاب کنید. (بالا را ببین)
  5. روی ذخیره کلیک کنید.

img

قالب بندی

اینها فیلدهای مورد نیاز و الزامات اولیه هستند. برای بررسی جزئیات قالب بندی اینجا را کلیک کنید.

tag_id (یا case_ID) یک شناسه منحصر به فرد (از کاراکترهای الفبای عددی استفاده کنید، خط فاصله پذیرفته می شود)

case_status 0 = بسته 1 = فعال

date_opened month/day/year mm/dd/yyyy

در مورد یک توصیف منحصر به فرد از مورد (استفاده از نویسه های الفبایی عددی، خط فاصله پذیرفته می شود)

img

در حال پردازش

بسته به تعداد رکوردهای موجود در فایل و بارگذاری سیستم، پلتفرم ممکن است چندین دقیقه طول بکشد تا وارد کردن را پردازش کند.

img

وقتی وارد کردن کامل شد، پلتفرم سوابق موردی را تنظیم می‌کند (یعنی اضافه یا اصلاح می‌کند) در داشبورد Cases سرپرست.

شکست

اگر عملیات واردات ناموفق باشد، پلتفرم یک پیام Failed را برمی‌گرداند. در بیشتر موارد، علت خرابی قالب بندی داده ها است.

img

برای دیدن علت شکست:

  1. در زیر عنوان ACTION روی View کلیک کنید.
  2. در صفحه بعد، ماوس را روی نماد Failed قرار دهید. راهنمای ابزار خطا را نمایش می دهد.
  3. به فایل CVS بروید و خطا را تنظیم و رفع کنید.
  4. فایل را دوباره آپلود کنید.
