چگونه ادمین ها کاربر ایجاد می کنند

سرپرست‌ها افرادی را که می‌توانند به برنامه‌ها و موارد دسترسی داشته باشند، ایجاد، تغییر یا غیرفعال می‌کنند. هنگامی که یک کاربر برای اولین بار وارد پلتفرم می شود، باید یک گردش کار نمایه را تکمیل کند.

  • برای اطلاعات بیشتر در مورد گردش کار نمایه کاربر جدید اینجا را کلیک کنید.

ایجاد کاربر

جدول زیر پنج راه برای ایجاد کاربر را نشان می دهد. برای یادگیری نحوه کار روی پیوند کلیک کنید.

| روش | نوع | اقدامات | جزئیات | کاربر جدید اعلان دریافت می کند؟ |
|----------------------------------------|------- ------|------------------------------------------ ------------------------|

استفاده از رابط کاربری | رابط کاربری | افزودن و اصلاح | دکمه مدیریت > کاربران | نه |

در حین ایجاد کیس و اختصاص کاربر به کیس | رابط کاربری | اضافه کردن | دکمه Admin > Cases > New button > Create Case page | بله، در صورت فعال بودن |

ایجاد خودکار کاربر با استفاده از لینک ثبت نام عمومی | اقدام کاربر | اضافه کردن | دکمه مدیریت > تنظیمات > صفحه راه اندازی سیستم | بله، اگر کیس ایجاد خودکار فعال باشد |

ابزار واردات | رابط کاربری | افزودن و اصلاح | دکمه Admin > Integrations > Imports | نه |

استفاده از API (به زودی) | API | افزودن و اصلاح | na | na |

اطلاعیه

این پلتفرم زمانی که کاربران به برنامه اضافه می شوند، اعلان ارسال نمی کند. با این حال، مدیران می‌توانند اعلان‌ها را برای تکالیف پرونده فعال کنند.
