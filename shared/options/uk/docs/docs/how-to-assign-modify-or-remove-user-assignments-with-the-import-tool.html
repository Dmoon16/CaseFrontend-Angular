Як призначити, змінити або видалити призначення користувача за допомогою інструмента імпорту

У цій статті адміністраторам показано, як керувати користувачами запитів за допомогою інструмента імпорту.

Адміністратори можуть:

  • призначити користувачів справі
  • змінювати користувачів, уже призначених для справи
  • видалити користувачів, призначених для справи

Відкриття інструмента імпорту

  1. Увійдіть в обліковий запис адміністратора.
  2. Знайдіть програму на сторінці програм і натисніть кнопку АДМІНІСТРАТОР.
  3. Клацніть вкладку «Інтеграція», а потім «ІМПОРТ» у підменю.
  4. Натисніть кнопку НОВИЙ. Відобразиться діалогове вікно «Додати імпорт».

Створення записів

Завантажте один або кілька файлів CSV. Використовуйте ці файли, щоб додавати, змінювати або видаляти користувачів регістру. Після відкриття діалогового вікна «Додати імпорт»:

  1. Натисніть поле Тип.
  2. Прокрутіть спадне меню та виберіть опцію (див. нижче). Відобразиться посилання.
  3. Натисніть посилання Завантажити зразок. Файл CSV завантажується на ваш пристрій.

малюнок

Відкрийте файл і додайте записи. Натисніть тут, щоб дізнатися, як використовувати дані формату у файлі CSV.

Параметри файлів

Призначити користувачів до випадків * Виберіть цей параметр, щоб додати користувачів до справи. * Залиште клітинку case_role_order пустою. * Перейдіть на сторінку «Користувачі» та скопіюйте ідентифікатор тегу для цієї особи. Якщо поле «Ідентифікатор мітки» пусте, введіть унікальний ідентифікатор, натисніть «Зберегти», а потім скопіюйте ідентифікатор у файл CSV.

Завантаження файлу

Ви можете завантажити файл одразу або повернутися до діалогового вікна «Додати імпорт» пізніше та виконати ці дії.

  1. Відкрийте діалогове вікно «Додати імпорт».
  2. Введіть унікальну назву для файлу CSV.
  3. Натисніть значок плюса (+), щоб завантажити файл зі свого пристрою.
  4. Клацніть поле Тип і виберіть потрібний параметр. (Дивись вище)
  5. Натисніть ЗБЕРЕГТИ.

Обробка

Залежно від кількості записів у файлі та завантаження системи платформі може знадобитися кілька хвилин, щоб обробити імпорт.

малюнок

Після завершення імпорту платформа коригує записи користувача (тобто додає, змінює або видаляє) на сторінці «Інциденти».

провал

Якщо операція імпорту не вдається, платформа повертає повідомлення Не вдалося. У більшості випадків причиною збою є форматування даних.

малюнок

Щоб побачити причину збою:

  1. Натисніть Переглянути під заголовком Дія.
  2. На наступній сторінці наведіть курсор миші на піктограму «Не вдалося». Підказка відображає помилку.
  3. Перейдіть до файлу CVS і виправте помилку.
  4. Завантажте файл ще раз.
