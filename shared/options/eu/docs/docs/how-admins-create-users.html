Administratzaileek erabiltzaileak nola sortzen dituzten

Administratzaileek sortu, aldatu edo desgaitu nor atzi dezaketen aplikazioak eta kasuak. Erabiltzaile bat plataforman lehen aldiz saioa hasten denean, profilaren lan-fluxu bat osatu behar du.

  • Egin klik hemen erabiltzailearen profil berriaren lan-fluxuari buruz gehiago jakiteko.

Erabiltzaile bat sortzea

Beheko taulak erabiltzaile bat sortzeko bost modu erakusten ditu. Egin klik estekan nola ikasteko.

| Metodoa | Mota | Ekintzak | Xehetasunak | Erabiltzaile berriak jakinarazpen bat jasotzen al duzu? |
|-------------------------|----------------|------- ------|--------------------------------------------- ------------------------|

UI erabiliz | UI | gehitu eta aldatu | Admin botoia > Erabiltzaileak | Ez |

Kasu bat sortzen duzun bitartean eta erabiltzailea kasuari esleitu | UI | gehitu | Admin botoia > Kasuak > Berria botoia > Sortu kasua orria | Bai, gaituta badago |

Sortu automatikoki erabiltzailea erregistratzeko esteka publikoa erabiliz | Erabiltzailearen ekintza | gehitu | Admin botoia > Ezarpenak > Sistemaren konfigurazio orria | Bai, sorkuntza automatikoko maiuskulak gaituta badago |

Inportatzeko tresna | UI | gehitu eta aldatu | Admin botoia > Integrazioak > Inportazioak | Ez |

APIa erabiliz (laster) | API | gehitu eta aldatu | na | na |

Jakinarazpenak

Plataformak ez die erabiltzaileei jakinarazpenik bidaltzen aplikazio batera gehitzen direnean. Hala ere, administratzaileek kasuen esleipenetarako jakinarazpenak gaitu ditzakete.
