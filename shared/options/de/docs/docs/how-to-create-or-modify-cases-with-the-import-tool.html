So erstellen oder ändern Sie Fälle mit dem Import-Tool

Dieser Artikel zeigt Administratoren, wie sie vorhandene Fälle mit dem Import-Tool erstellen oder ändern.

Öffnen des Import-Tools

  1. Melden Sie sich beim Administratorkonto an.
  2. Suchen Sie eine App auf der Seite Apps und klicken Sie auf die Schaltfläche ADMIN.
  3. Klicken Sie im Admin-Bereich auf die Registerkarte Integration und dann im Untermenü auf IMPORTS.
  4. Klicken Sie auf die Schaltfläche NEU. Das Dialogfeld „Importe hinzufügen“ wird in einer neuen Browser-Registerkarte angezeigt.

Bild

Fälle erstellen

Nachdem das Dialogfeld Importe hinzufügen geöffnet wurde:

  1. Klicken Sie auf das Feld Typ und wählen Sie eine Option aus (siehe unten). Ein Link wird angezeigt.
  2. Klicken Sie auf den Link Beispiel herunterladen. Eine CSV-Datei wird auf Ihr Gerät heruntergeladen.

Als nächstes öffnen Sie die Datei und fügen Fälle hinzu.

Typ

Wenn Sie auf das Feld Typ klicken, wird ein Dropdown-Menü angezeigt. Wählen Sie eine von zwei Optionen für den Bulk-Case-Import aus.

  • Hinweis Das Dropdown-Menü enthält viele Optionen, aber nur zwei beziehen sich auf die Fallverwaltung. Jede Option hat eine andere Beispiel-CSV-Datei.

Fälle hinzufügen

  • Wählen Sie diese Option für einen einfachen Bulk-Upload von Fällen.
  • Klicken Sie auf den Link Beispiel herunterladen, um eine CSV-Datei mit Spaltenüberschriften für den Importtyp Fälle erstellen zu öffnen.

Fälle ändern

  • Wählen Sie diese Option, um aktuelle Fälle zu aktualisieren.
  • Klicken Sie auf den Link Beispiel herunterladen, um eine CSV-Datei mit Spaltenüberschriften für den Importtyp Fälle ändern zu öffnen.

Hochladen der Datei

Sie können eine Datei sofort hochladen oder zu einem späteren Zeitpunkt zum Dialogfeld „Importe hinzufügen“ zurückkehren und diese Schritte ausführen.

  1. Öffnen Sie das Dialogfeld „Importe hinzufügen“.
  2. Geben Sie einen eindeutigen Namen für die CSV-Datei ein.
  3. Klicken Sie auf das Pluszeichen (+), um die Datei von Ihrem Gerät hochzuladen.
  4. Klicken Sie auf das Feld Typ und wählen Sie eine Option aus. (siehe oben)
  5. Klicken Sie auf SPEICHERN.

Formatierung

Dies sind die Pflichtfelder und ersten Voraussetzungen. Klicken Sie hier , um die Formatierungsdetails zu überprüfen.

tag_id (oder case_ID) eine eindeutige Kennung (verwenden Sie alphanumerische Zeichen, Bindestriche werden akzeptiert)

case_status 0 = geschlossen 1 = aktiv

date_opened Monat/Tag/Jahr MM/TT/JJJJ

über eine eindeutige Beschreibung des Falls (alphanumerische Zeichen verwenden, Bindestriche werden akzeptiert)

Bild

Wenn der Import abgeschlossen ist, passt die Plattform Falldatensätze (dh fügt hinzu oder ändert) auf dem Fall-Dashboard des Administrators an.

Fehler

Wenn der Importvorgang fehlschlägt, gibt die Plattform eine Fehlernachricht zurück. In den meisten Fällen ist die Fehlerursache die Datenformatierung.

Bild

So sehen Sie die Fehlerursache:

  1. Klicken Sie unter der Überschrift AKTION auf Ansicht.
  2. Bewegen Sie auf der nächsten Seite die Maus über das Symbol „Fehlgeschlagen“. Der Tooltip zeigt den Fehler an.
  3. Gehen Sie zur CVS-Datei und passen Sie den Fehler an und beheben Sie ihn.
  4. Laden Sie die Datei erneut hoch.
