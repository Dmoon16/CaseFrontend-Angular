Paano Gumawa o Magbago ng Mga Case gamit ang Tool sa Pag-import

Ipinapakita ng artikulong ito sa mga admin kung paano gumawa o magbago ng mga kasalukuyang kaso gamit ang tool sa Pag-import.

Binubuksan ang tool sa Pag-import

  1. Mag-log in sa admin account.
  2. Maghanap ng app sa page ng Apps at i-click ang ADMIN button.
  3. Sa Admin panel, i-click ang tab na Integration at pagkatapos ay IMPORTS sa submenu.
  4. I-click ang BAGONG button. Ang dialog box na Magdagdag ng Mga Pag-import ay nagpapakita sa isang bagong tab ng browser.

img

Paglikha ng mga kaso

Pagkatapos magbukas ng dialog box ng Add Imports:

  1. I-click ang field na Uri at pumili ng opsyon (tingnan sa ibaba). Nagpapakita ang isang link.
  2. I-click ang link na I-download ang Sample. Nagda-download ang isang CSV file sa iyong device.

Susunod, buksan ang file at magdagdag ng mga case.

Uri

Kapag na-click mo ang field ng Uri , magpapakita ang isang drop-down na menu. Pumili ng isa sa dalawang opsyon para sa maramihang pag-import ng case.

  • Tandaan Ang drop-down na menu ay may maraming mga opsyon, ngunit dalawa lang ang nauugnay sa pamamahala ng kaso. Ang bawat opsyon ay may ibang sample na CSV file.

Magdagdag ng mga kaso

  • Piliin ang opsyong ito para sa isang simpleng maramihang pag-upload ng mga case.
  • I-click ang link na I-download ang Sample upang magbukas ng CSV file na may mga heading ng column para sa uri ng pag-import ng Create cases.

Baguhin ang mga kaso

  • Piliin ang opsyong ito para i-update ang mga kasalukuyang kaso.
  • I-click ang link na I-download ang Sample upang magbukas ng CSV file na may mga heading ng column para sa uri ng pag-import ng Modify cases.

Pag-upload ng file

Maaari kang mag-upload kaagad ng file o bumalik sa dialog box na Magdagdag ng Mga Pag-import sa ibang araw at kumpletuhin ang mga hakbang na ito.

  1. Buksan ang dialog box na Magdagdag ng Mga Import.
  2. Maglagay ng natatanging pangalan para sa CSV file.
  3. I-click ang icon na plus (+) upang i-upload ang file mula sa iyong device.
  4. I-click ang field na Uri at pumili ng opsyon. (tingnan sa itaas)
  5. I-click ang I-SAVE.

img

Pag-format

Ito ang mga kinakailangang field at unang kinakailangan. Mag-click dito upang suriin ang mga detalye ng pag-format.

tag_id (o case_ID) isang natatanging identifier (gumamit ng mga alphanumeric na character, tinatanggap ang mga gitling)

case_status 0 = sarado 1 = aktibo

petsa_bukas buwan/araw/taon mm/dd/yyyy

tungkol sa isang natatanging paglalarawan ng kaso (gumamit ng mga alphanumeric na character, tinatanggap ang mga gitling)

img

Pinoproseso

Depende sa bilang ng mga tala sa pag-load ng file at system, maaaring tumagal ng ilang minuto ang platform upang maproseso ang pag-import.

img

Kapag nakumpleto ang pag-import, inaayos ng platform ang mga talaan ng kaso (ibig sabihin, nagdaragdag o nagbabago) sa dashboard ng Mga Kaso ng admin.

Kabiguan

Kung nabigo ang pagpapatakbo ng pag-import, magbabalik ang platform ng isang Nabigong mensahe. Sa karamihan ng mga kaso, ang sanhi ng pagkabigo ay ang pag-format ng data.

Upang makita ang sanhi ng pagkabigo:

  1. I-click ang View sa ilalim ng ACTION heading.
  2. Sa susunod na pahina, i-hover ang mouse sa icon na Nabigo. Ipinapakita ng tooltip ang error.
  3. Pumunta sa CVS file at ayusin at ayusin ang error.
  4. I-upload muli ang file.
