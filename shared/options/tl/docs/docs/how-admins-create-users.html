Paano Gumagawa ang Mga Admin ng Mga User

Ang mga admin ay gumagawa, nagbabago o nagdi-disable kung sino ang makaka-access ng mga app at case. Kapag nag-log in ang isang user sa platform sa unang pagkakataon, kailangan nilang kumpletuhin ang workflow ng profile.

  • Mag-click dito para matuto pa tungkol sa bagong workflow ng profile ng user.

Paglikha ng isang gumagamit

Ang talahanayan sa ibaba ay nagpapakita ng limang paraan upang lumikha ng isang user. Mag-click sa isang link upang matutunan kung paano.

| Paraan | Uri | Mga Aksyon | Mga Detalye | Nakatanggap ng notification ang bagong user? |
|------------------------|----------------|------- ---------------------|--------------------|--------------------- ------------------------|

Gamit ang UI | UI | magdagdag at baguhin | Button ng admin > Mga User | Hindi |

Habang gumagawa ng case at italaga ang user sa case | UI | magdagdag ng | Pindutan ng Admin > Mga Case > Button na Bagong > Lumikha ng pahina ng Case | Oo, kung pinagana |

Awtomatikong lumikha ng user gamit ang pampublikong link sa pag-signup | Aksyon ng user | magdagdag ng | Button ng admin > Mga Setting > page ng System Setup | Oo, kung ang auto create case ay pinagana |

Tool sa pag-import | UI | magdagdag at baguhin | Pindutan ng Admin > Mga Pagsasama > Mga Pag-import | Hindi |

Gamit ang API (paparating na) | API | magdagdag at baguhin | na | na |

Mga abiso

Ang platform ay hindi nagpapadala ng notification sa mga user kapag idinagdag sila sa isang app. Gayunpaman, maaaring i-enable ng mga admin ang mga notification para sa mga pagtatalaga ng kaso.
