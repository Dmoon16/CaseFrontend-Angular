Comment les administrateurs créent des utilisateurs

Les administrateurs créent, modifient ou désactivent qui peut accéder aux applications et aux cas. Lorsqu'un utilisateur se connecte à la plateforme pour la première fois, il doit compléter un workflow de profil.

  • Cliquez ici pour en savoir plus sur le nouveau workflow de profil utilisateur.

Création d'un utilisateur

Le tableau ci-dessous montre cinq manières de créer un utilisateur. Cliquez sur un lien pour savoir comment.

| Méthode | Taper | Gestes | Détails | Le nouvel utilisateur reçoit une notification ? |
|-------------------------|---------------|------- ------|--------------------|---------------------- ------------------------|

Utilisation de l'interface utilisateur | UI | ajouter & modifier | Bouton Admin > Utilisateurs | Non |

Lors de la création d'un cas et de l'affectation d'un utilisateur au cas | UI | ajouter | Bouton Admin > Requêtes > bouton Nouveau > page Créer une requête | Oui, si activé |

Création automatique d'un utilisateur à l'aide d'un lien d'inscription public | Action de l'utilisateur | ajouter | Bouton Admin > Paramètres > page Configuration du système | Oui, si la création automatique de cas est activée |

Outil d'importation | UI | ajouter & modifier | Bouton Admin > Intégrations > Importations | Non |

Utilisation de l'API (à venir) | API | ajouter & modifier | na | na |

Avis

La plateforme n'envoie pas de notification aux utilisateurs lorsqu'ils sont ajoutés à une application. Cependant, les administrateurs peuvent activer les notifications pour les attributions de cas.
